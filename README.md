# Linux System Programming

본 레포지토리는 컴퓨터학부 전공 과목 Linux System Programming에서 수행한 실습 과제(Lab)들을 정리한 저장소이다.

Linux 환경에서의 시스템 프로그래밍을 중심으로  
파일 시스템, 프로세스 제어, 데몬 프로세스, 시스템 콜, 파일 시스템 내부 구조(ext2)를 단계적으로 구현하며 학습하였다.

각 Lab은 단순 기능 구현이 아닌,  
**운영체제가 제공하는 추상화 계층 아래에서 실제로 무엇이 일어나는지**를  
직접 구현하고 확인하는 것을 목표로 한다.


---

## Labs Overview

### Lab01 – SSU Cleanup
Linux 환경에서 동작하는 사용자 정의 쉘 프로그램을 구현하였다.

디렉토리 구조를 트리 형태로 출력하고,  
디렉토리 내 파일을 확장자 기준으로 정리하는 기능을 제공한다.

이 실습을 통해 파일 시스템 탐색,  
시스템 콜 기반 파일 처리, 프로세스 제어의 기초를 학습하였다.

주요 학습 요소:
- 디렉토리 순회 및 재귀 구조
- 파일 정보 처리(stat, lstat)
- fork / exec 기반 프로세스 실행
- 사용자 입력 처리 및 명령 분기


### Lab02 – SSU Cleanupd
지정한 디렉토리를 지속적으로 모니터링하며  
자동으로 파일을 정리하는 **데몬 프로세스**를 구현하였다.

백그라운드 서비스 구조를 직접 설계하고,  
시그널과 설정 파일, 로그 관리를 통해  
실제 시스템 서비스에 가까운 형태로 확장하였다.

주요 학습 요소:
- 데몬 프로세스 생성 (double-fork, setsid)
- 시그널 처리(SIGHUP, SIGTERM)
- 설정 파일 및 로그 파일 관리
- 디렉토리 변경 감지 및 자동 정리 로직


### Lab03 – SSU-EXT2
ext2 파일 시스템 디스크 이미지 파일을 직접 파싱하여  
파일 시스템 내부 구조를 탐색하는 쉘 프로그램을 구현하였다.

운영체제의 파일 API를 사용하지 않고,  
on-disk 구조체를 직접 정의하여  
inode, block, directory entry를 해석하였다.

주요 학습 요소:
- ext2 슈퍼블록 및 그룹 디스크립터 파싱
- inode 기반 디렉토리 및 파일 탐색
- Direct / Indirect block 구조 이해
- 파일 시스템 메타데이터 구조 분석

## Learning Outcomes

이 레포지토리를 통해 다음과 같은 역량을 학습하고 구현하였다.

- Linux 시스템 콜 기반 프로그래밍
- 파일 시스템 구조 및 동작 원리 이해
- 프로세스 생성, 종료, 시그널 처리
- 데몬 프로세스 설계 및 관리
- 파일 시스템 내부(ext2) 구조 직접 파싱

단순히 라이브러리를 사용하는 수준을 넘어,  
**운영체제 내부 동작을 코드로 설명할 수 있는 수준**을 목표로 한다.
